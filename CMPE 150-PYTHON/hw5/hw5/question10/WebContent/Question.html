Question 10:

Calculation Parser

Please read important information first.

Description

In this problem, you need to implement the calc_parser(calc_str) function recursively. This function should return result of the expression written on the string calc_str

The format for the calc_str and any other nested expressions in it:

  • Either + or - sign will be used for any of the calculations.
  • There will always be a single whitespace both before and after a + or - sign.
  • On either side of the + or - sign, there will either be a positive integer number or another expression (that follows the same format as calc_str). In the case it's an expression, it will be covered in parentheses.
  • The outermost expression will not be covered in parentheses. Meaning that user will enter (1 - 2) + (1 + 2) instead of ((1 - 2) + (1 + 2))

Input taking part is handled for you, predefined variables are:

  • inp_str: A non-empty string that follows the format described above.

Output printing part is handled for you as well, you only need to implement calc_parser(calc_str) function recursively..

Following are true for every question:

  • Important Note 1: If you don't implement it recursively, your grade will be 0 even if your outputs are correct.
  • Important Note 2: Inside your recursive functions, you are not allowed to use:
    • Variables that have global keyword
    • Variables that are defined outside the recursive function (unless specified otherwise)

------------------------------------------------------------------

Check the examples for further clarification. Keep in mind that we will be grading your code not just based on these examples, but other cases as well, so try to write code which can handle all possible cases.

Examples:
INPUT OUTPUT
(1 + 2) - ((22 - 34) + ((1 - 3) + 32)) -15
(13 - (1 + 2)) + (11 + 23) 44
5 - (1 + 3) 1
